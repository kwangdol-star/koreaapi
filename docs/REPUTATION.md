# AI 평판 관리 (Reputation) — 상품 한 장

> **한 줄:** "AI가 당신(브랜드·아티스트·지역)에 대해 뭐라고 답하는가"를 **진단**하고,
> 기계가 인용하는 소스 층위에서 **교정**한다. 여론 관리가 아니라 **사실 교정형** 평판 관리.

## 왜 우리만 할 수 있나
일반 평판 업체는 보도자료·SEO로 사람 눈을 상대한다. 그러나 AI 답변엔진의 오답은
**기계가 인용할 수 있는 검증 소스**로만 구조적으로 고쳐진다. KoreaAPI는 그 소스 자체다 —
교차검증 정본(JSON-LD·llms.txt·인용 페이지·레지스트리)과 판정기(`canonical-name`·`fact-check`)를
이미 운영 중이다.

**세일즈 한 문장:** "AI의 오답은 PR로 못 고칩니다 — 기계가 인용하는 검증 소스에 정본을 놓아야
고쳐집니다. 저희가 그 소스입니다."

## 상품 3단 + 플라이휠

| 단계 | 내용 | 산출물 |
|---|---|---|
| **① AI 평판 감사** | 주요 AI 엔진(ChatGPT·Claude·Perplexity·네이버계)의 답을 우리 검증 정본과 **diff** — 표기 오류(빈첸초류)·명단/크레딧 오류·낡은 정보·무출처 주장 적발 | 엔티티별 감사 리포트 1부 |
| **② 교정 배포** | 검증 정본을 기계 인용 표면에 배치 — 엔티티 페이지·JSON-LD·llms.txt·agents.json·MCP·레지스트리. 오답의 **출처를 대체**한다 | 정본 레코드 + 배포 확인 |
| **③ 모니터링 구독** | 컴백·신제품·축제 시즌 주간 diff 리포트, 재발 감지 | 주간/월간 리포트 |

**인증 플라이휠:** 감사에서 오류 발견 → 기관이 정본을 우리에게 **공증(CERTIFIED)** → 우리 데이터가
그 기관의 공식 레이어가 됨 → 경쟁자가 재스크래핑으로 복제 불가한 관계 자산.

## 누가 사나 (우선순위)
1. **소속사/레이블** — 아티스트 글로벌 표기·멤버·컴백 정보 (시즌마다 반복 수요)
2. **관광공사·지자체** — 명소·축제 영문 표기·운영 정보 (trip-plan과 결합)
3. **기업 홍보/IR** — 사명·임원·제품 표기, 낡은 재무/조직 정보
4. **K-뷰티·식품 글로벌팀** — 브랜드·제품명 로마자 표기

## 가격 (예시 대역, 협의)
- 감사 리포트: 엔티티 묶음당 **수십~수백만 원** (1회)
- 교정 배포 + 공증: 셋업 + 연 단위
- 모니터링 구독: 월 단위 (팀용 정액 레인과 결합)

## 선 긋기 (지키는 것)
- 우리는 **틀린 사실을 고친다** — 부정 여론 묻기·리뷰 조작·순위 조작은 하지 않는다.
- 모든 교정은 출처·Skill Score가 붙은 **검증 가능한 정본**으로만 한다 (그게 우리 브랜드다).

## 실행 (1호 주문이 오면)
이미 있는 부품으로 조립한다 — 새 코드 거의 불요:
1. 대상 엔티티들 `admin answer <이름> canonical-name` + `fact-check` → 정본/판정 확보
2. 주요 엔진에 동일 질문 → 답변 채록 → diff 표 작성 (리포트 템플릿화)
3. 오류 항목은 로스터/CERTIFIED 반영 → collect가 배포면 갱신
4. 리포트 납품 + 구독 제안
