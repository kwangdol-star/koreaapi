# KoreaAPI — 결제(x402) 활성화 가이드

## TL;DR
x402 결제 레일은 **이미 완성**돼 있고 **기본은 꺼짐(dormant)** 입니다. 받을 **지갑 주소 하나**만
설정하면 켜집니다. 켜지면 프리미엄 엔드포인트(`/v1/korea-rising`)를 AI 에이전트가 **호출당 USDC로
자동 결제**합니다 — 계정·API 키 불필요. 개인키는 서버에 넣지 않습니다.

## 왜 이 구조인가 (형제 KWeather와 다른 선택)
- **형제(KWeather)** = *자가정산(self-settling) 릴레이어*. 개인키가 든 hot wallet + gas를 직접 운영해
  온체인 정산을 스스로 함. 형제는 이미 릴레이어 인프라를 돌리기 때문에 가능한 것.
- **KoreaAPI** = *facilitator(키리스)*. 온체인 크립토(서명 검증·전송)는 외부 facilitator가 처리하고,
  **우리는 개인키를 절대 쥐지 않는다.** 받을 지갑 **주소만** 필요.
- 1인·비개발 운영에는 키리스가 **더 안전**(키 탈취 = 자금 손실 위험 없음)하고 운영 부담이 없다.
  자금은 우리 지갑 주소로 직접 정산된다. → KoreaAPI는 키리스를 유지한다.

## 켜는 법

### 1단계 — 지갑 준비 (받을 주소만)
EVM 지갑(MetaMask 등)을 만들고 **주소(`0x...`)** 만 확보. 개인키는 어디에도 넣지 않는다.

### 2단계 — 테스트넷으로 먼저 (리스크 0)
API 호스트(Render 등)에 환경변수 설정:
```
X402_PAY_TO   = 0x<받을_지갑주소>
X402_NETWORK  = base-sepolia      # 테스트넷 (기본값)
X402_PRICE_USD = 0.01             # 호출당 가격(USD)
```
→ `/v1/korea-rising`이 HTTP 402를 반환하고, 에이전트가 테스트넷 USDC로 결제하면 통과.
확인: `GET /healthz` → `"x402_active": true`.

### 3단계 — 메인넷 (실제 수익)
```
X402_PAY_TO         = 0x<받을_지갑주소>
X402_NETWORK        = base                    # Base 메인넷 (실제 USDC)
X402_FACILITATOR_URL = <메인넷 facilitator>    # 예: Coinbase CDP facilitator
X402_PRICE_USD      = 0.01
```
- 메인넷은 **실제 USDC를 정산하는 facilitator**가 필요(Coinbase CDP 등). 기본 facilitator는
  테스트넷(base-sepolia) 전용이다.

## 끄기
`X402_PAY_TO`를 지우면 즉시 dormant(프리미엄 무료 제공)로 복귀. 안전 기본값.

## 무엇이 무료이고 무엇이 과금인가
- **무료(계속 개방):** 검증 데이터 전부 — `/v1/verified`, `/v1/resolve`, `/v1/answer`(Answer Products),
  `/latest.json`, MCP 도구. AEO/인용 권위를 위해 일부러 열어 둔다.
- **과금:** `/v1/korea-rising`(독점 수요 신호) — 호출당 x402.
- **협의 레인 (준비됨):** 검증 데이터셋의 LLM 학습/평가용 라이선스 — 해외 선례(Wiley·Informa 등
  "쌓인 문서" 라이선스)의 KoreaAPI 버전. 정본 표기 정답셋은 한국어 모델 eval 재료로 그대로 쓰인다.
- 확장: 다른 프리미엄 엔드포인트도 `api.py`의 `_gate()`로 감싸면 같은 방식으로 과금.

## 마켓플레이스 연결
"에이전트가 계정 없이 **호출당 자동 결제(x402)**" = 정부 AI 에이전트 마켓플레이스가 지향하는
에이전트-네이티브 결제에 부합. **키리스**라 운영 리스크도 낮다 — 입점 소개(`docs/MARKETPLACE.md`)의
가격 항목과 함께 제시.

## 현실 점검 (언제 실제로 돈이 되나)
레일은 준비됐지만, x402 수익은 **결제하는 에이전트가 실제로 존재**해야 발생한다. 따라서 순서는:
1. (지금) 데이터·Answer Products·검증 권위를 키워 **인용/호출되는 표면**을 넓힌다.
2. 마켓플레이스 입점 등으로 **에이전트 트래픽**을 확보한다.
3. 트래픽이 붙으면 위 3단계로 **메인넷 x402를 켠다** → 호출당 정산.

즉 레일은 "켤 준비 완료" 상태로 두고, 우선순위는 **표면 확장 → 트래픽 → 과금**이다.

## 돈 그림 (정직) — 어디서 버는가, 그리고 언제
데이터 접근 자체엔 값을 못 붙인다 — Wikidata라는 공짜 대체재가 옆에 있어서 **API 구독은 제일 약한 레인**이다. 진짜 돈은 데이터 위에 얹는 세 갈래:

1. **커머스 수수료 (제일 큰 베팅)** — `get_buy_options`의 "공식 맞아? 검증 → 구매" 흐름. 에이전트가
   대신 결제하는 시대가 오면 K-컬처 티켓·앨범·굿즈 거래액의 **bps**만 먹어도 데이터 판매와 자릿수가
   다르다. x402 같은 에이전트 결제 레일이 깔리면 per-call 과금도 자연스러워진다. → 코드상 `buy_options`를
   *검증-공식 게이트 + 수수료-레디 인텐트 원장*으로 심어둠(레일 dormant, 0 bps).
2. **공급측 B2B (후행)** — 소속사·기획사가 "공식 정정권 + 검증 배지"에 월 구독하는 블루체크 모델
   (`CERTIFIED` 티어). 단 **인용 점유율이 먼저** 생겨야 팔리는 후행 매출.
3. **답변엔진 라이선스** — Perplexity류에 한국 문화 팩트 레이어를 통째로. 가능하나 바이어 소수·세일즈
   사이클 김.

**정직한 타임라인:** 향후 ~1년은 매출 거의 0 + 신선도 그라인드 구간. 이건 데이터 장사가 아니라
**"에이전트 커머스가 온다"에 베팅해 신뢰 레이어 자리를 선점하는 포지션 게임**이다. 파도가 오면
수수료업으로 크게 벌고, 안 오면 잘 만든 무료 유틸리티로 남는다. **돈이 되냐 → 되는데, 후불이다.**

→ 그래서 지금 우선순위는 매출이 아니라 ①인용 점유율(citation share) ②신선도 그라인드 ③시간 이력
축적. 셋이 서야 위 세 레인이 순서대로 켜진다.
